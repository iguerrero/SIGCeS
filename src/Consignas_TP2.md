## Trabajo Práctico N°2

### Objetivo

Esta actividad permite reforzar los conocimientos del lenguaje de programación *Java* 
mediante el desarrollo de una aplicación que da solución a los requerimientos de una situación problemática.

El propósito del trabajo práctico de la materia es realizar una aplicación programada en Java 
que permita resolver una situación problemática a libre elección. Esto te permitirá aplicar 
en forma individual los conocimientos desarrollados en la materia en los distintos módulos: 
modelo orientado a objetos, operadores, control de flujo, herencia, polimorfismo, encapsulamiento, arreglos, etc.
### Situación Problemática

Para resolver esta actividad deberás retomar la situación problemática planteada en el **Trabajo Práctico 1**.

### Consigna

En la primera instancia del trabajo práctico se definió una situación problemática a resolver. En esta segunda instancia se debe comenzar con la programación de la resolución de la situación problemática utilizando el lenguaje de programación Java y los conceptos de orientación a objetos aprendidos hasta el momento. Se debe desarrollar una aplicación que permita resolver los requerimientos planteados en la instancia anterior. Para el desarrollo de la aplicación se puede utilizar cualquier IDE de programación como por ejemplo Eclipse o NetBeans.

- Se debe desarrollar un proyecto utilizando el lenguaje de programación Java que permita cumplimentar con los requerimientos que los alumnos han planteado en la instancia anterior del presente trabajo práctico. Es decir, anteriormente se definió una situación problemática a resolver junto a las entidades que participan, y en esta instancia se debe desarrollar el programa que permita resolver dicha situación problemática.

Se requiere:

- Crear un proyecto base en Java.
- Codificar las clases que se corresponden a las entidades identificadas en la primera instancia del trabajo práctico.
- Implementar el código Java que da resolución a la situación problemática.

El programa debe incluir la interacción con el usuario mediante el ingreso de datos por teclado, 
por ejemplo, mediante un menú de opciones. Se deben emplear paquetes que permitan una mejor organización de las distintas clases del proyecto Java.

La entrega de esta actividad es individual y cada alumno debe tomar como base la situación problemática 
planteada en forma grupal en la instancia anterior.

Para la codificación del proyecto se puede utilizar cualquier IDE de programación, 
por ejemplo, Eclipse o NetBeans. 
El proyecto debe ser entregado sin errores de compilación, ordenado, con formatos de 
presentación acordes y se pueden utilizar comentarios que permitan entender algún aspecto del código.

El programa desarrollado por el alumno en esta instancia del trabajo práctico permitirá, en las instancias 
siguientes, la aplicación de los siguientes conceptos de programación:

- Trabajo práctico 3: herencia, redefinición de métodos, clases abstractas, arreglos.
- Trabajo práctico 4: polimorfismo, excepciones, interfaces.

### Formato Entregable

El alumno, en forma individual, debe entregar un archivo comprimido en formato __.zip__ 
que contiene las carpetas y archivos fuentes del proyecto (archivos de extensión .java 
dentro de las carpetas correspondientes a los paquetes del proyecto). Por ejemplo, si la 
aplicación fue desarrollada utilizando la IDE de programación **Eclipse** o **NetBeans**, 
se debe entregar los archivos fuentes que se encuentran dentro del directorio “src” tomando 
como referencia el directorio base del proyecto.

El proyecto no debe tener errores de compilación y el código debe seguir un formato y orden 
legible. El proyecto debe ser entregado de forma tal que su ejecución no tenga inconvenientes 
luego de importar el código fuente.

El nombre del archivo a entregar debe seguir el siguiente formato:

*POO_<año>_TP2_<nombre proyecto>_<apellido alumno>.zip*

Ejemplo:

*POO_2021_TP2_GestionAlumnos_Perez.zip*
